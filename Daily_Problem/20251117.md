# [C - Boring Day](https://codeforces.com/contest/1982/problem/C)

## 解题思路

1. 最直接的贪心想法是每一轮抽取的牌点数之和刚好在范围$[l, r]$内时结束本轮. 如果点数之和大于$r$, 先丢牌让点数和小于$r$(把丢弃的牌当成一个回合),  如果此时点数和大于$l$结束本轮, 如果小于$l$继续抽牌, 不断的重复上面的操作直到点数在$[l, r]$之内才结束当前回合
1. 上面的贪心是否能够得到最优解呢? 最优解之中赢的回合总是不可再分的(不能够划分成两个满足条件的回合), 在上述贪心想法中一定不存在能够划分的回合

## 参考代码


```cpp
// 算法时间复杂度O(n)
#include <bits/stdc++.h>
using namespace std;

#define endl '\n'
#define rep(i, a, b) for (int i = a; i <= b; ++i)
#define erp(i, a, b) for (int i = a; i >= b; --i)

using i64 = long long;
using i32 = int;
using pii = pair<int, int>;
using pll = pair<i64, i64>;

const int mod = 1000000007, mod0 = 998244353;

void solve() {
    int n, mi, mx;
    cin >> n >> mi >> mx;

    vector<int> a(n + 1);
    rep (i, 1, n)
        cin >> a[i];    
    int ans = 0, j = 1;
    i64 sum = 0;
    rep (i, 1, n) {
        sum += a[i];
        while (j <= i && sum > mx) {
            sum -= a[j];
            ++j;
        }
        if (sum >= mi) {
            ++ans;
            sum = 0;
            j = i + 1;
        }
    }
    cout << ans << endl;
}

int main() {
    ios::sync_with_stdio(0);
    cin.tie(0);
    cout.tie(0);
    int t;
    cin >> t;
    while (t-- > 0)
        solve();
    return 0;
}
```
